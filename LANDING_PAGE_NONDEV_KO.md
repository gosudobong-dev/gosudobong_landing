# 고수의 운전면허 도봉점 랜딩페이지 비개발자 설명서

이 문서는 “개발 지식 없이” 페이지의 역할과 흐름을 이해할 수 있도록 설명합니다.

## 1) 이 랜딩페이지의 목적
- 방문자가 “여기서 상담/예약하고 싶다”는 마음을 갖게 만드는 것
- 전화, 카카오톡, 네이버 예약 같은 행동으로 자연스럽게 이어지게 하는 것

## 2) 이 페이지가 설득하려는 사람들
- 빨리 면허가 필요한 사람 (속성)
- 장롱면허로 불안한 사람 (심리/안전)
- 운전 공식이 필요한 사람 (기능/도로주행)
- 특정 부분만 연습하고 싶은 사람 (핀셋 연습)
- 가격을 따져보는 사람 (가성비)

## 3) 첫 화면에서 하는 일
- 가장 강한 메시지와 큰 버튼을 보여줍니다.
- “나한테 맞는 과정이 있겠다”는 확신을 주는 역할입니다.

## 4) 핵심 설득 장치(참여형 도구)
코스별 페이지에는 짧은 참여형 도구가 바로 나옵니다. 이게 방문자 관심을 붙잡는 핵심입니다.

- 속성: “언제 면허를 딸 수 있는지” 날짜 계산
- 공식: “운전 DNA” 테스트
- 장롱면허: “불안감 처방전” 체크리스트
- 연습장: “나만의 커리큘럼” 설계
- 가성비: “비용 비교 계산기”

이 도구들은 “재미 + 나에게 맞춤” 느낌을 줘서 상담/예약 확률을 올립니다.

## 5) 신뢰를 만드는 구간
- 후기: 실제 후기를 보여줌
- 위치: 지도와 방문 정보 제공
- FAQ: 자주 묻는 질문으로 불안 해소
- 이벤트: 당장 움직이게 만드는 촉진 요소

## 6) 행동 유도(전환) 구조
상담/예약 행동을 쉽게 하도록 여러 위치에 버튼이 반복됩니다.
- 상단 버튼: 바로 전화/상담
- 중간 CTA: 코스 소개 뒤 바로 제안
- 하단 떠다니는 버튼: 어디서든 상담 가능

## 7) 클릭 후 사용자가 기대하는 흐름
- 전화 연결 또는 카카오톡 상담으로 이어짐
- 네이버 예약으로 이어지는 경우도 있음
- 이 흐름이 끊기지 않게 “문구와 버튼 목적”이 일관되어야 함

## 8) 보완하면 좋은 점 (비개발자 관점)

1. 메뉴를 눌러도 원하는 곳으로 안 내려가는 경우
- 상단 메뉴가 특정 위치로 이동하려는데, 실제로는 이동이 안 되는 페이지가 있음
- 고객 입장에서는 “페이지가 고장났다”로 느낄 수 있음

2. 메인 페이지의 “아래로 내려가기” 표시가 맞지 않을 수 있음
- 화살표를 눌러도 이동이 안 되는 경우가 있어 초반 경험이 끊김

3. 후기 이미지 설명이 너무 길어 접근성이 떨어짐
- 화면 읽기 기능(시각 보조)에 불필요한 긴 문장이 들어감

4. 가격/부가세 표기가 페이지마다 다름
- 어떤 곳은 “부가세 포함”, 어떤 곳은 “별도”로 보임
- 고객에게 가격이 불명확하게 보일 수 있음

5. 진단/테스트 결과 문구가 비슷하거나 반복됨
- “맞춤형” 느낌이 약해질 수 있음

6. 과한 움직임(애니메이션)이 산만하게 느껴질 수 있음
- 배지나 강조 요소가 계속 튀면 집중이 흐트러질 수 있음

## 9) 운영/관리 관점 체크리스트
- 광고 문구가 바뀌면 페이지 전체에서 일관되게 보여야 함
- CTA(상담/예약) 버튼 링크가 하나라도 빠지면 전환이 크게 떨어짐
- 후기/가격/이벤트는 신뢰 요소이므로 최신 상태 유지 필요
